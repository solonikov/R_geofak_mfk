Цель настоящего задания --- закрепление навыков создания тематических карт в
среде R средствами пакета tmap. Во вложении к заданию находится архив data.zip,
содержащий часть слоев, необходимых для построения карт.

Напишите скрипт, который формирует следующие карты:

1. Карта «Климатические пояса» (пример — Plot_1.png).

Источники данных:
- climates.gpkg (в приложенном архиве)
- данные Natural Earth масштаба 1:110M (через пакет rnaturalearth)

Проекция: V проекция Вагнера (Wagner V)
Градусная сетка: имеется
Масштабная линейка: отсутствует

Слои карты:
- Океан (ocean)
- Реки (rivers_lake_centerlines)
- Озера (lakes)
- Климатические пояса (climates.gpkg)
- Лини тропиков и полярных кругов (geographic_lines)

Подсказки:
- Для задания проекции выясните ее строку PROJ.4:
https://proj4.org/operations/projections/index.html

- Цвета климатических поясов можно взять из задания по геоинформатике для 2
курса:
https://tsamsonov.github.io/arcgis-course/map-design-climates.html#map-design-climates-thematic

- Слои географической основы следует оформить в серых цветах
- Климатические пояса отображаются с прозрачностью
- Для упорядочения климатических поясов от Арктического к Экваториальному вам
необходимо превратить соответствующий столбец фрейма данных в упорядоченный
фактор перед построением карты.

2. Карта «Изменения площади леса в эстуарии реки Конго (2000-2012 гг)» (пример
— Plot_2.png).

Источники данных:
- растровые покрытия *.tif (в приложенном архиве), получены по данным
GlobalForestChange
(https://earthenginepartners.appspot.com/science-2013-global-forest)

Проекция: градусная (преобразование не требуется)
Градусная сетка: отсутствует
Масштабная линейка: присутствует

Слои карты:
- % площади, покрытой лесом на 2000 год (trees.tif)
- Обезлесение к 2012 году (loss.tif)
- Прирост леса к 2012 году (gain.tif)
- Водная поверхность (mask.tif)

Подсказки:
- Для размещения сведений об источниках данных и масштабной линейки используйте
функции tm_credits() и tm_scale_bar() соответственно
- Растр loss.tif содежит значения от 0 до 17, где 0 -- это фоновое значение, а
1-17 кодируют пикселы, в которых обнаружено обезлесение в соответствующий год
(2001-2017).
- Растр gain.tif содержит значения 0 и 1, где 0 -- это фоновое значение, а 1
кодирует пикселы, в которых обнаружено появление леса к 2012 году.
- Растр mask.tif содержит значения 0, 1 и 2, где 0 -- это отсутствие
данных, 1 -- картографируемая площадь, 2 -- вода

Чтобы наложить обезлесение и прирост леса поверх растра 2000 года, вам
необходимо сделать пустыми (NA) все пикселы, не относящиеся к периоду
2001-2012 гг и приросту леса соответственно. Далее оставшимся пикселам присвоить
значение 1. Аналогично накладывается растр водной поверхности. При отображении
этих растров используйте палитру из одного цвета, для растра водной поверхности
легенду следует убрать.

3. Карта «Протяженность и электрификация железных дорог, 2016 год» (пример
-- Plot_3.png)

Источники данных:
- Таблица «Railway transport - length of lines, by nature of transport»
Евростата (пакет eurostat)
- Географические данные единиц NUTS уровня 0 -- страны (пакет eurostat)

Подробнее про NUTS:
https://ec.europa.eu/eurostat/web/nuts/background

Проекция: коническая равнопромежуточная (Equidistant Conic) с центральным
меридианом 20 в.д. и стандартными параллелями 43 с.ш. и 62 с.ш.
Градусная сетка: отсутствует
Масштабная линейка: отсутствует

Слои карты:
- Протяженность железных дорог, км
- Электрификация, %

Подсказки:
- Для задания проекции выясните ее строку PROJ.4:
https://proj4.org/operations/projections/index.html
- Чтобы выгрузить нужную таблицу Евростата, найдите ее код по названию
- Отфильтруйте данные на 2016 год
- Для подсчета доли электрифицированного транспорта вам потребуется выполнить
пару преобразований и фильтрацию таблицы (используйте возможности tidyr и dplyr)
- Чтобы выгрузить границы стран, используйте функцию get_eurostat_geospatial() c
параметром nuts_level = '0' (соответствует странам)
- Для присоединения таблицы к географическим данным используйте код в поле geo
(доступен до применения label_eurostat).
- Чтобы разместить кружки четко внутри стран, вы можете получить
репрезентативную точку для каждого региона посредством функции
sf::st_point_on_surface(), и далее применять tm_bubbles() к точкам. Точку лучше
получать по непроецированным регионам, и затем уже проецировать.
- Чтобы ограничить охват карты, используйте параметр bbox функции tm_shape()
