Задание №6 посвящено освоению базовых навыков построения графиков средствами
системы ggplot. Работа выполняется на примере данных USDA по почвенным сериям.
Ваша задача — визуализировать данные по физико-географическим условия
формирования четырех почвенных серий. Краткие сведения про пакет soilDB и
сервис Soil Data Access, необходимые для выполнения работы, содержатся в
6-й лекции.

Названия почвенных серий: cecil, altavista, lloyd, wickham

Условия формирования:
— положение на склоне (hillpos),
— материнская порода (pmorigin),
— годовое распределение осадков (climate.monthly)

Инструкции по выполнению задания:
1. Загрузите данные по указанным сериям с использованием пакета soilDB.
2. Постройте столбчатый график эмпирических вероятностей нахождения почв данной
серии на разных положениях по склону (таблица hillpos, пример Plot_6f_1.png).
3. Постройте круговые секторные диаграммы эмпирических вероятностей нахождения
почв данной серии на разных материнских породах (таблица pmorigin, пример
Plot_6f_2.png).
4. Постройте линейный график, характеризующий среднемноголетний внутригодовой
ход осадков для почв данной серии (таблица climate.monthly, переменная Potential
(поле Varible), значения q50 — медиана, пример Plot_6f_3.png

Подсказки:
— необходимые таблицы содержатся в списке, который возвращает функция
fetchOSD(). Информация о том, как извлекать объекты их списков, содержится в
соответствющем разделе курса:
https://tsamsonov.github.io/r-geo-course/data-structures.html#lists
— для того чтобы отрисовать позиции по склону в нужном порядке, необходимо их
превратить в упорядоченный фактор. По умолчанию будет применяться цветовая шкала
viridis. Чтобы инвертировать последовательность цветов в ней, добавьте к графику
функцию scale_fill_viridis_d(direction = -1).
— для раскрашивания круговой диаграммы хорошо различимыми цветами можно
использовать функцию scale_fill_manual(...), и передать ей в параметр values
вектор с цветам для разных горных пород. Используйте пакет RColorbrewer, чтобы
сформировать такой вектор.

Форма представления:
— файл, названный латиницей по шаблону 06_Фамилия.R (например, 06_Samsonov.R)

Требования:
— соблюдение общепринятого стиля программирования,
— осмысленные названия переменных (только латиница),
— комментарии по ходу программы
— редактировать входные данные и названия файлов нельзя
— организация необходимых преобразований данных посредством конвейера
манипуляций
